Resumen de la aplicación

Zync es una plataforma móvil orientada al entretenimiento nocturno que conecta a tres tipos de participantes en un mismo ecosistema: asistentes a eventos, DJs y dueños de negocios/venues. La aplicación permite gestionar pedidos en bar, solicitar canciones a DJs, chatear en tiempo real dentro del evento y administrar códigos promocionales, gigs y lineups.

🎵

Solicitud de canciones

Los asistentes buscan canciones en Spotify y las solicitan al DJ en tiempo real, con precio por canción definido por el propio artista.

💬

Chat en tiempo real

Chat grupal del evento y chat 1-a-1 entre usuarios. Indicadores de escritura, estado de entrega y visto.

🛒

Pedidos y pagos

Carrito de compra, códigos promo y sistema de puntos Zync redimibles como saldo ($1 = 1 punto).

🎭

Multi-rol

Un solo usuario puede ser asistente, DJ y organizador simultáneamente, cambiando de rol desde el perfil.

📱

QR Code

Escáner QR para pagos en venue y check-in de asistentes en los eventos.

🎟️

Gestión de eventos

Creación de lineups, asignación de DJs a eventos, historias de eventos y presencia en tiempo real.

Roles de usuario

Cada usuario puede tener uno o más roles activos. El rol activo determina qué navegación y funcionalidades ve.

🙋

User (Asistente)

  • Feed principal del evento
  • Solicitar canciones al DJ
  • Carrito de pedidos en bar
  • Chat grupal del evento
  • Billetera y Zync Points
  • Scanner QR para pagar
🎧

DJ (Artista)

  • Ver solicitudes de canciones
  • Gestionar códigos promo
  • Ver gigs confirmados
  • Editar perfil artístico
  • Banner, logo, redes sociales
  • Perfil público visible
🏢

Business (Venue)

  • Gestionar productos/menú
  • Scanner QR entrada/pedidos
  • Crear lineups de eventos
  • Asignar DJs a eventos
  • Administrar venues
  • Ver solicitudes del evento

Cambio de rol

El componente RoleSelector en la pantalla de perfil permite al usuario cambiar su rol activo instantáneamente. Para activar el rol DJ, primero debe crear su perfil de artista. Para el rol business, debe crear su perfil de organizador.

Backend

API REST + WebSocket en http://44.222.141.70:3000. Autenticación JWT con Bearer token. El token se almacena en expo-secure-store (móvil) o localStorage (web) con caché en memoria para evitar lecturas repetidas.

Gestión de estado

Solo Context API de React, sin Redux ni Zustand. Cuatro contextos globales:

  • AuthContext — estado de autenticación, usuario, tokens, balance
  • RoleContext — rol activo, cambio de rol
  • ZyncContext — establecimiento/venue activo seleccionado
  • CartContext — carrito, cantidades, checkout, órdenes activas

Chat en tiempo real

El sistema de chat usa Socket.io para mensajería en tiempo real. Hay dos modalidades: chat grupal del evento y chat privado 1-a-1.

💬 Chat grupal del evento src/features/chat/screens/GroupChatScreen.tsx
  • Header con título "CHAT GENERAL", número de usuarios online y estado de conexión
  • Carrusel de historias del evento en la parte superior
  • Carrusel de usuarios conectados con sus avatares
  • Historial completo de mensajes del evento
  • Burbujas propias (gradiente neon) y ajenas (dark glass con avatar y nombre)
  • Timestamp en cada mensaje
  • Indicador de escritura animado cuando alguien escribe
  • Iconos de estado del mensaje: enviado / entregado / visto
  • Auto-scroll al último mensaje
  • Marca mensajes como "vistos" automáticamente
  • Input con blur + botón de envío (deshabilitado si no hay conexión)
🔒 Chat privado 1-a-1 src/features/chat/screens/SingleChatScreen.tsx
  • Header con avatar y nombre del otro usuario + indicador online/offline
  • Misma estructura de burbujas y estados que el chat grupal
  • Indicador de escritura específico del otro usuario
  • Fondo con imagen y overlay gradiente
  • Historial completo de conversación
👥 Usuarios conectados src/features/chat/screens/ConnectedUsersScreen.tsx
  • Lista de todos los usuarios activos en el evento
  • Avatar, nombre y badge "En línea" con punto verde

Sistema de Zync Points

  • Los usuarios acumulan puntos con cada compra
  • 1 punto equivale a $1 de descuento al hacer checkout
  • Saldo de puntos visible en Wallet y Perfil
  • Toggle en carrito para activar/desactivar uso de puntos

Códigos Promocionales

  • Los DJs generan códigos asociados a un evento específico
  • El asistente ingresa el código en el carrito
  • El sistema redime el código via redeemPromoCode(code)
  • El contador de uso del código se incrementa en cada redención

Integraciones externas

Spotify Web API

Usada en la pantalla Beats para buscar canciones que los asistentes pueden solicitar al DJ.

  • Flujo Client Credentials OAuth (sin login del usuario)
  • Token cacheado con expiración automática
  • Endpoint: GET /search?q={query}&type=track
  • Retorna: título, artista, URL del artwork del álbum, ID de track
  • Credenciales almacenadas en .env (CLIENT_ID, CLIENT_SECRET)

Socket.io (Backend propio)

  • Conexión autenticada via JWT en el campo auth del handshake
  • Salas por evento (join-event con eventId)
  • Mensajería grupal y privada en tiempo real
  • Sistema de presencia con lista de usuarios online
  • Confirmaciones de entrega y lectura

expo-image-picker

  • Selección de imágenes de galería o cámara
  • Usado para avatar de usuario, logo DJ y banner DJ
  • Subida como FormData multipart al backend
