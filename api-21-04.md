# Zync API — Endpoints en uso (21-04-2026)

Base URL: `EXPO_PUBLIC_API_URL` (ej. `http://100.31.134.251:3000`)  
Auth header: `Authorization: Bearer <accessToken>`  
🔒 = requiere auth | 👤 = solo el propio usuario | 🎪 = solo ORGANIZER | 🎧 = solo DJ | 🏢 = solo dueño del venue

## WebSocket (Socket.IO)

Conectar a `ws://<HOST>:3000?token=<accessToken>`

### Cliente → Servidor

| Evento | Payload | Descripción |
|--------|---------|-------------|
| `event:join` | `{ eventId }` | Unirse a sala del evento |
| `event:leave` | `{ eventId }` | Salir de sala del evento |
| `chat:send_public` | `{ eventId, content }` | Mensaje grupal |
| `chat:send_private` | `{ eventId, toUserId, content }` | Mensaje privado |
| `presence:get_list` | `{ eventId }` | Pedir lista de usuarios presentes |
| `chat:typing` | `{ eventId, toUserId }` | Indicar que estás escribiendo |
| `chat:mark_delivered` | `{ messageId }` | Marcar mensaje como entregado |
| `chat:mark_seen` | `{ messageId }` | Marcar mensaje como visto |

### Servidor → Cliente

| Evento | Payload | Descripción |
|--------|---------|-------------|
| `chat:public_message` | `{ id, userId, content, sentAt }` | Nuevo mensaje grupal |
| `chat:private_message` | `{ id, fromUserId, content, sentAt }` | Nuevo mensaje privado |
| `presence:list` | `[{ userId, ... }]` | Lista de usuarios presentes |
| `presence:joined` | `{ userId }` | Usuario entró al evento |
| `presence:left` | `{ userId }` | Usuario salió del evento |
| `chat:typing_status` | `{ fromUserId, eventId }` | El otro está escribiendo |
| `order:new` | objeto orden completo | Business recibe nuevo pedido |
| `order:status_update` | `{ orderId, status }` | User recibe cambio de estado de pedido |
| `song_request:updated` | objeto song request completo | User recibe respuesta del DJ |

## Roles

| Rol | Navegación app | Cómo se asigna |
|-----|---------------|----------------|
| `USER` | `(tabs)/` | Default al registrarse |
| `DJ` | `(business)/` | Al crear DJ profile |
| `ORGANIZER` | `(business)/` | Al crear organizer profile |
| `STAFF` | `(business)/` | Asignado por organizador |

Un usuario puede tener múltiples roles simultáneamente.

## Formato de errores

```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "...",
  "errorCode": "DJ_PROFILE_NOT_FOUND",
  "timestamp": "2026-04-21T..."
}
```

Errores de validación incluyen array `errors: [{ field, message }]`.  
Errores 500 incluyen `requestId` para correlacionar con logs del servidor.
